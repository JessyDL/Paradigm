meta::library class

container class for meta::file's

detailed description

a meta::library is a collection of meta::file instances. The library will do the basic bookkeeping, allowing you to search and retrieve meta::file instances based on their UID, or tags and narrow it down depending on the meta::file type as well.

There are also facilities to have UID's reference other UID's, and do bi-directional lookups for these relations. As well as the ability to load the persistent companion file that the meta::file describes (in case the UID satisfies is_physical_file() ), and to have that companion file cached for faster reloads.

library (psl::string8::view lib)

description

location on disk where the library can be found.

parameters
 in lib The filepath to which file should be loaded. This path can either be absolute or relative.

detailed description

The constructor will try to load the given filepath, and then parse it. It will also create the minimal representation of each meta::file entry in the given file, with its tags, etc..

bool serialize (const UID& uid)

description

serializes the given UID to disk in case it is present, and an on-disk file.

parameters
 in uid The UID you wish to serialize.
returns true when the UID is both present in the library, and it is linked to an on-disk file.

detailed description

When you send a UID to this method, the meta::library will verify that it has an entry with the given UID, and that it is linked to an on-disk location. When it satisfies those two requirements, it will then proceed to serialize the file to disk using the current values that are present.

bool remove (const UID& uid, bool safe_mode = true)

description

tries to remove the given UID from the library.

parameters
 in uid The UID you wish to remove.
 in safe_mode when true (default), will disallow removing as long as this item is being referenced by others.
returns true if the instance was removed or not.

detailed description

Will try to remove the given UID from the meta::library. On success it will notify all other meta::file's that it references, that they no longer are being referenced by this instance.

safe_mode controls wether you should avoid deleting this instance if others are still referencing this UID (true), or forcibly delete this instance regardless (false).

std::unordered_set<UID> referencing (const UID& uid) const

description

gets a list of all UID's the given UID might be referencing.

parameters
 in uid the UID to find all references for.
returns a list of all UID's the given UID is referencing.

detailed description

as an example, a core::gfx::material could be referencing a core::gfx::texture, this means that if you call this method using the core::gfx::material's UID, then you will get a list back that atleast contains the core::gfx::texture UID.

Note

will return an empty list in case the given UID is not present in the meta::library.

std::optional<psl::string8::view> load (const UID& uid)

description

loads the specific UID's content.

parameters
 in uid the given UID to optionally find the content for.
returns the resulting content in UTF-8 format if found.

detailed description

Loads the specified UID's associated file, not meta::file, and returns it (if found). The associated file is the companion file for which the meta::file was generated, for example a shader will have a core::meta::shader file on disk, but also the actual shader file (SPIR-V for Vulkan). This returns the SPIR-V file. The file content will get cached into the meta::library to make subsequent load() calls faster.

Warning

this method requires the file to satisfy is_physical_file(), otherwise it will silently fail.

bool unload (const UID& uid)

description

purges the specific UID's cachec content.

parameters
 in uid the given UID to purge from the cache.
returns true if it was found on the meta::library, and had cached contents.

detailed description

Unloads the specified UID's associated file, not meta::file, and returns true if it was in the cache and purged. The associated file is the companion file for which the meta::file was generated, for example a shader will have a core::meta::shader file on disk, but also the actual shader file (SPIR-V for Vulkan). This returns the SPIR-V file. You can reload the file into the cache by calling load() with the given UID as parameter.

Warning

this method requires the file to satisfy is_physical_file(), otherwise it will silently fail.

template<typename T = file>
std::optional<T*> get (const UID& uid) const

description

get the UID's associated meta::file from the meta::library and tries to cast to the templated type.

template parameters
T should satisfy at least being either meta::file or being derived from meta::file
parameters
 in uid the given UID to find.
returns the meta::file, or derived class associated with the UID.

detailed description

Searches the meta::library for the given UID, and then tries to satisfy the templated type given. If successful it will return the given template type (default meta::file), otherwise it returns an std::nullopt in case the conversion is not possible at runtime, and will give a compile error when the template type does not satisfy the type constraint.

Warning

this method cannot downcast, it has to satisfy the type perfectly.
